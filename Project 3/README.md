# SQL Sales Data Analysis with PostgreSQL

## Project Overview

This project demonstrates the use of SQL to analyze a retail sales dataset and answer business-related questions. Using PostgreSQL and pgAdmin 4, SQL queries were written to retrieve, filter, group, aggregate, and sort data, transforming raw transactional records into meaningful business insights.

The project focuses on applying fundamental SQL concepts commonly used in data analytics and business intelligence.

---

## Objectives

- Import a cleaned dataset into PostgreSQL.
- Retrieve data using SQL queries.
- Filter records using the `WHERE` clause.
- Group data using `GROUP BY`.
- Perform aggregations using SQL functions.
- Sort query results using `ORDER BY`.
- Answer business questions through data analysis.

## Business Questions Answered

1. How many total orders are in the dataset?
2. How many unique customers made purchases?
3. What is the total revenue generated?
4. What is the average order value?
5. Which product generated the highest sales?
6. Which product generated the lowest sales?
7. Which payment method was used most frequently?
8. Which referral source generated the most orders?
9. Which payment method generated the highest revenue?
10. Which customers spent the most money?
11. Which product sold the highest quantities?
12. What are top 10 highest value orders?
13. How many orders belong to each order status?
14. Which orders used coupon codes?
15. What is the average quantity purchased per order and per product?

---

## Key Findings

- **Total Orders:** 1,200
- **Unique Customers:** 1,189
- **Total Revenue:** 1,264,761.96
- **Average Order Value:** 1,053.97
- **Highest Selling Product:** Chair
- **Lowest Selling Product:** Phone
- **Most Frequently Used Payment Method:** Online
- **Highest Revenue Payment Method:** Credit Card
- **Top Referral Source:** Instagram
